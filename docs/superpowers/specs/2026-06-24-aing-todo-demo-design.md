## Mục Đích

Reference implementation của AING methodology: **semantic model là single source of truth** dùng chung cho cả human review và AI code generation. Khách hàng clone project này về, thay domain của họ vào `semantic/semantic_model.yaml`, chạy generator, rồi để AI viết code từ semantic + spec.

**Hai mục tiêu song song:**
- **A — Showcase:** Demo pipeline AING cho khách hàng xem cách làm đúng
- **C — Starter kit:** Template clone về và thay domain

---

## Pipeline 4 Bước

```
[Human]  spec/spec.md
[Human]  semantic/semantic_model.yaml
         ↓
[Script] python generate.py  →  generated/ddl/schema.sql
                             →  generated/dbt/schema.yml
                             →  generated/pydantic_models.py
         ↓
[AI]     đọc spec + semantic  →  app/ (routes, services, db)
[Human]  review spec + semantic ONLY — không review code
         ↓
[CI]     dbt test  →  quality gate tự động
```

## `generate.py` — Mapping Rules

Script đọc `semantic/semantic_model.yaml`, sinh ra 3 artifacts. Chạy: `python generate.py` — overwrite toàn bộ `generated/`, in diff để phát hiện migration cần thiết.

### → `generated/ddl/schema.sql`

| YAML field | SQL output |
|-----------|-----------|
| `attributes.type` | column type (uuid→UUID, string→VARCHAR, timestamp→TIMESTAMPTZ, boolean→BOOLEAN, enum→VARCHAR + CHECK) |
| `attributes.nullable: false` | `NOT NULL` |
| `attributes.default` | `DEFAULT value` |
| `attributes.max_length` | `VARCHAR(n)` |
| `attributes.unique: true` | `UNIQUE` |
| `attributes.references` | `REFERENCES table(id)` |
| `tests.unique` | `UNIQUE` constraint |
| `tests.not_null` | enforced via `NOT NULL` |
| enum `values` | `CHECK (col IN ('a','b','c'))` |

### → `dbt/models/schema.yml` (generated, DO NOT EDIT)

| YAML field | dbt test |
|-----------|---------|
| `tests.not_null` | `- not_null` |
| `tests.unique` | `- unique` |
| enum `values` | `- accepted_values: values: [...]` |
| `attributes.references` | `- relationships: to: ref('...') field: id` |
| `tests.custom` | custom SQL test block, `expect: empty` |

### → `generated/pydantic_models.py`

| YAML field | Pydantic output |
|-----------|----------------|
| `attributes.type` | Python type annotation |
| `attributes.nullable: true` | `Optional[...]` |
| `attributes.default` | `Field(default=...)` |
| `attributes.max_length` | `Field(max_length=n)` |
| enum `values` | `Literal['a', 'b', 'c']` |
| `business_rules` | docstring trong class |

## `app/` — FastAPI Layer

### Nguyên tắc bắt buộc (AI phải tuân theo)

1. `routers/` chỉ parse HTTP → gọi service → trả response. Không chứa business logic.
2. `services/` enforce business rules với comment trích dẫn ID: `# BR-T01: completed todo không được sửa title`
3. `app/schemas.py` import từ `generated/pydantic_models.py`, không tự định nghĩa lại.
4. Raw SQL với psycopg2 — không dùng ORM.

### Phân chia không sinh được (AI làm, script không làm)

- Status transition logic: `status → completed` thì set `completed_at = NOW()`
- Authorization: collaborator vs owner permissions
- Cascade logic: archive list → soft-delete toàn bộ todo trong list

## `dbt/` — Test Runner

dbt chỉ chạy tests, không transform data. Input là `generated/dbt/schema.yml` (generated từ semantic model).

```bash
dbt test                    # chạy toàn bộ
dbt test --select todos     # chỉ test một model
dbt test --store-failures   # lưu failing rows để debug
```

CI gate: chạy sau mỗi deploy và hàng đêm. Fail = alert, không deploy tiếp.
